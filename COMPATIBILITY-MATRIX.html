1. Critical Research Findings

Finding 1: Ubuntu Noble NEVER Updated DCN 3.1.5 DMCUB Firmware

Confirmed from Ubuntu Noble changelog through v0ubuntu2.26 (26 point releases). The changelog updates DCN 3.1.4, 3.1.6, 3.5, 3.5.1, 4.0.1 — but DCN 3.1.5 is never mentioned. The Debian fix (Bug #1057656, firmware-nonfree 20240709-1) was never backported to Ubuntu Noble.

Finding 3: MODE2 Reset Does NOT Reset DCN — mode0 Might Break the Crash Loop

GPU IP Block Routing on Raphael:

GCHUB ← GC (Graphics Compute) + SDMA ← MODE2 resets ONLY THESE
MMHUB ← VCN + JPEG + VPE ← MODE1 resets this
DCHUB ← DCN (Display Core Next) ← NOT TOUCHED BY MODE2

Why the crash loops:
1. DCN stalls (optc31 timeout)
2. GFX ring hangs (gnome-shell hit stalled DCN)
3. MODE2 reset → restores GFX, but DCN still broken
4. gnome-shell re-submits → hits broken DCN again → repeat

Fix: amdgpu.reset_method=1 (mode0 = full ASIC reset)
Resets ALL IP blocks including DCHUB/DCN. Novel, untested on Raphael.

Finding 4: simpledrm Card Ordering Fix Confirmed

initcall_blacklist=simpledrm_platform_driver_init is the confirmed fix from Arch Linux and community blogs. simpledrm grabs card0 before any GPU driver loads. Blocking it lets amdgpu claim card0.

Warning: Removes early boot console (Plymouth, LUKS prompt). Display is black until amdgpu loads. Acceptable for workstation without disk encryption.

Finding 5: GNOME Ring Timeout Is Cross-Distro

Reported on Fedora 42, Ubuntu 25.04, Ubuntu 24.04. Switching distros without switching compositors will NOT fix this. XFCE or Sway required if GNOME keeps crashing.

Finding 6: NVIDIA 595 — Open Modules Now Default, modeset=1 Now Default

Key changes: Open kernel modules installed by default. nvidia-drm.modeset=1 enabled by default (remove from GRUB if already set to avoid doubling). CudaNoStablePerfLimit for full P0 clock in CUDA. Build fix for kernel 6.19. xfwm4 compositor blinking fix included.

Why Kernel 6.17 Still Crashed Previously

Kernel 6.17 has ALL DCN31 patches. The failures were caused by:

  1. Firmware still ancient (HIGHEST) — .bin.zst priority means manual update was ignored. DMCUB 0x05002F00 is pre-fix.
  2. simpledrm steals card0 — GNOME picks wrong GPU.
  3. Test A stripped all params — sg_display=-1, ppfeaturemask wrong.
  4. Stale initramfs — Not rebuilt after kernel switch.

Conclusion: Kernel 6.17 is correct. The failures were firmware + config, not kernel.

Why GNOME Crashes but XFCE Won't

The crash loop requires TWO conditions: (1) optc31 DCN stall at boot, AND (2) GFX ring submissions from compositor. XFCE's xfwm4 with XRender generates zero GFX ring commands. Even if DCN stalls, XFCE never triggers the ring timeout that starts the loop.

11. Documentation Gaps & Untested Parameters

Critical Gaps

  • sg_display=0 stripped by Test A, listed as CRITICAL in 5 docs
  • ppfeaturemask running value 0xfff7bfff ≠ documented 0xfffd7fff
  • reset_method never mentioned in ANY existing doc
  • DCN 3.1.5 SRU status never checked — all docs assumed Ubuntu updated it

Never Tested Parameters

ML Workstation Autoinstall Variants

Dual-GPU crash loop isolation testing — AMD Ryzen 9 7950X iGPU + NVIDIA RTX 4090

Ryzen 9 7950X (Zen 4) RTX 4090 (AD102) ASUS ROG X670E Hero Ubuntu 24.04 LTS Kernel 6.17 HWE

The Problem

The ML workstation suffers from an intermittent boot crash loop caused by a two-condition failure model:

ConditionDescriptionEvidence
1. DCN Pipeline Stall optc31_disable_crtc REG_WAIT timeout during EFI-to-amdgpu handoff. DMCUB firmware 0.0.15.0 is critically outdated (pre-dates all known fixes). dmesg at 6.1s every crash boot
2. GFX Ring Pressure Compositor (gnome-shell) submits GL commands to GFX ring. Ring hangs on corrupted DCN state. MODE2 reset only fixes GFX/SDMA, NOT DCN. ring gfx_0.0.0 timeout always from gnome-shell PID

Remove either condition = stable system. That's the testing strategy: variants progressively isolate and fix each condition.

  • If FAIL: need to check Xorg.0.log, try AccelMethod "none"
  • Requires USB firmware preparation step

Critical Finding

This is the most important variant. Every variant after B depends on this result. DMCUB 0.0.15.0 is version 0.0.15.0 — it predates even the initial community-reported firmware in linux-firmware.git (0.0.88.0). The July 2024 Debian fix (0.0.224.0+) repaired the DMCUB state machine that manages CRTC disable/enable transitions. Without this fix, the OTG_BUSY flag stays stuck, causing the optc31 timeout cascade.

Advantages

  • Most modern, polished UI — full GNOME desktop
  • Full NVIDIA + CUDA + Docker ML stack
  • Wayland session (modern display protocol)
  • Best app ecosystem (Nautilus, Extensions, Settings GUI)
  • Production target if firmware fix works
  • Ubuntu's default DE — best support/documentation
  • gnome-tweaks + extension-manager for customization
  • Overview/Activities for window management

Disadvantages

  • HIGHEST crash risk — Mutter GL compositing = max GFX ring pressure
  • gnome-shell was the crash trigger in every previous failure
  • Mutter RT thread SIGKILL bug requires workaround
  • Animations disabled for safety (less smooth feel)
  • Highest RAM usage (~1.5GB+)
  • Most complex configuration (GDM3, dconf, services)
  • If GNOME fails, fallback to D/E/F required
  • NVIDIA 580 from repos, not latest 595

Critical Finding

Variant G is the "endgame" test. If it passes with firmware ≥0.0.255.0, the system can return to full Ubuntu GNOME — the most productive desktop for most users. The 7 mitigations (KMS thread type, HW cursors off, animations off, check-alive timeout, cursor blink off, lockup timeout, GDM hardening) create a safety net that should survive even marginal firmware improvements. If it still fails, the D/E/F fallback desktops are proven stable alternatives.
